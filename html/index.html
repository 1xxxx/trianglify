Trianglify

Trianglify generates colorful geometric patterns from random input. It was inspired by btmills' geopattern project, and can output patterns in a variety of formats. The initial version was written in a single day because I got fed up with Adobe Illustrator. The v0.2.0 release of Trianglify represents a ground-up rewrite which eliminates the dependency on d3.js and includes a brand new canvas rendering implementation.

Features

  • No external dependencies
  • GPL-licensed
  • SVG, PNG, and canvas output
  • Only 32kB minified

Getting Started

You can find officially maintained packages for Trianglify on bower, npm, and CDNJS.

Trianglify exports a function which takes a set of options as the argument. It returns an object with the pattern it generated and methods to render that pattern to SVG, PNG, or canvas.

// Basic Trianglify usage
var options = {cell_size: 30, x_colors: 'random'};
var pattern = Trianglify(options); // Generate a pattern
document.body.appendChild(pattern.canvas());  // Append to the document

Not a programmer? Check out the Trianglify wallpaper generator example to make and download patterns.

Output Format

The pattern object returned by Trianglify has the following properties/methods:

pattern.polys

An array of the polygon colors and polygons that make up the pattern

pattern.opts

The options set for this pattern (see the options section for details).

pattern.svg()

Render the pattern to SVG format and return it as a DOM node that can be appended to the document.

pattern.canvas([canvas])

Render the pattern to a canvas and return it as a DOM node that can be appended to the document. If a reference to an existing canvas is passed in as the canvas argument, Trianglify will render to it instead.

Options

Trianglify starts with the following values by default, which can be customized by passing in an object with the options you want to override.

var defaults = {
  cell_size: 75,
  variance: 0.75,
  x_colors: Trianglify.colorbrewer.RdYlBu,
  y_colors: 'match_x',
  palette: Trianglify.colorbrewer,
  color_space: 'lab'
  color_function: false,
  stroke_width: 1.51,
  width: 600,
  height: 400,
  seed: null
}

width

The desired width of the pattern in pixels.

height

The desired height of the pattern in pixels.

seed

A string or numeric seed for the random number generator. When this is set to null or undefined, the generator will be seeded automatically.

cell_size

This option controls the size of the cells that Trianglify uses to generate patterns. A bigger cell size will result in larger triangles in the output. This should be an integer value > 0. Use the slider below to see the effect this option has.

variance

This option can range from 0 to 1 and controls the variablity of the triangle vertices that Trianglify generates. 0 gives a very regular grid of triangles, while 1 gives a very irregular pattern. Use the slider below to see the effect this option has.

palette

This option allows you to change the set of color gradients that Trianglify chooses from when the 'random' option is set on either axis. It accepts an array of arrays. (While the example uses color keywords, all CSS color formats are supported).

color_space

This option changes the color space Trianglify uses for interpolation. See this article for a good explanation of why this matters.

color_function

This option allows you to specify your own function for coloring the pattern. It should take the x and y coordinates of the triangle centroid and return the color you want the triangle to have in a valid CSS color format.

stroke_width

This option allows you to specify the width of the stroke applied to the pattern. If you are using the canvas rendering mechanism, I recommend leaving it at it's default of 1.51, which will prevent ugly antialiasing artefacts.
